# Инструкции по исправлению метода findPrediction

## Проблема
Метод `findPrediction` обрезает текст прогноза, останавливаясь на символе "с" (из-за паттерна `[^с]+?`).

## Решение
Замените строки 485-486 в файле `server/parsers/predictionsParser.js`:

### Было:
```javascript
      /(?:Моя\s+ставка|Мой\s+прогноз):\s*([^с]+?)(?:\s+с\s+коэффициентом|\s+за\s+\d+\.\d+)/i,
      /(?:Моя\s+ставка|Мой\s+прогноз):\s*([^.]{1,200})/i,
```

### Должно быть:
```javascript
      /(?:Моя\s+ставка|Мой\s+прогноз)[^:]*:\s*([^.]*(?:\.[^.]*)*)/i,
      /(?:Моя\s+ставка|Мой\s+прогноз)[^:]*:\s*([^\n]+)/i,
```

Также удалите строку 495, которая обрезает текст:
```javascript
        predictionText = predictionText.replace(/\s+с\s+коэффициентом.*$/i, '').trim();
```

## Дополнительно
Для улучшения поиска всех прогнозов (чтобы находить все 12, а не только 2), нужно также улучшить логику в методе `parsePredictions`.
